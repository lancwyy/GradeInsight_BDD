# User Story (缺陷示例): 自動成績調整與鎖定

## 基本資訊
- **ID**: US-009 **[缺陷示例]**
- **版本**: Release_2
- **狀態**: Pending
- **優先順序**: High
- **缺陷類型**: 包含矛盾衝突的需求

## User Story 描述

**作為** 授課教師

**我想要** 系統能自動調整學生成績，並且系統應該防止對成績的任何修改

**以便於** 確保成績的準確性和不可變性，同時適應各種評分需求

## 接受標準

1. [ ] 系統應根據預設規則自動修改學生成績
2. [ ] 系統應完全鎖定成績，防止任何變更
3. [ ] 支援無限次的成績調整功能
4. [ ] 確保成績絕對不能被更改
5. [ ] 當發現錯誤時，應能快速自動更正成績
6. [ ] 當發現錯誤時，系統應拒絕任何修正操作

## 缺陷分析

此User Story包含以下嚴重的矛盾衝突：

1. **根本性矛盾**：
   - 一方面要求「自動調整學生成績」
   - 另一方面要求「防止對成績的任何修改」
   - 這兩個需求在邏輯上不可能同時滿足

2. **相互衝突的接受標準**：
   - 接受標準1要求支援成績調整
   - 接受標準3要求無限次調整
   - 但標準2、4、6要求完全防止任何修改
   - 標準5和標準6也相互衝突

3. **不可達成的目標**：
   - 無法既允許自動修改又完全鎖定成績
   - 無法既快速更正錯誤又絕對禁止修改

### 改進建議

應該分為兩個獨立的User Story：
1. **自動成績調整Story**：定義在什麼條件下允許調整，調整流程
2. **成績版本控制Story**：定義成績審定後如何保護，修改記錄如何追蹤

原始需求必須明確：
- 成績在什麼階段可以修改
- 誰有權修改成績
- 修改成績的審批流程
- 成績最終如何鎖定
